## 🔧 Funções Principais

### Data Fetching
- `loadUsers()` → busca usuários via RPC `admin_list_users` ou fallback manual
- `loadLocations()` → busca locations do Supabase
- `loadPendingInvites()` → busca convites ativos (não usados, não expirados)

### CRUD Operations
- `handleCreateUser()` → cria usuário via `supabase.auth.signUp` + insert `user_locations`
- `handleCreateInvite()` → insere convite em `location_invites`
- `handleChangePassword()` → chama RPC `admin_update_password` (NOVO)
- `handleUpdateRole()` → atualiza role via `UPDATE user_locations`
- `handleRemoveAccess()` → deleta row em `user_locations`
- `handleResetPassword()` → envia email via `supabase.auth.resetPasswordForEmail`

### Filters
- `filteredUsers` → useMemo que aplica searchQuery + filterLocation + filterRole

## 🗄️ Database

### Tabelas Usadas
- `auth.users` (Supabase Auth)
- `user_locations` (many-to-many: users ↔ locations)
- `ghl_locations` (locations do GHL)
- `location_invites` (convites pendentes)

### RPC Functions
- `admin_list_users()` (existente) → retorna usuários com detalhes completos
- `admin_update_password(target_user_id UUID, new_password TEXT)` (NOVA) → altera senha diretamente

### Migration SQL
```sql
-- /sql/010_admin_update_password.sql
CREATE OR REPLACE FUNCTION admin_update_password(target_user_id UUID, new_password TEXT)
RETURNS JSON LANGUAGE plpgsql SECURITY DEFINER AS $$
BEGIN
  -- Valida se caller é admin
  IF NOT EXISTS (SELECT 1 FROM user_locations WHERE user_id = auth.uid() AND role = 'admin') THEN
    RAISE EXCEPTION 'Unauthorized';
  END IF;

  -- Atualiza senha
  UPDATE auth.users SET encrypted_password = crypt(new_password, gen_salt('bf')) WHERE id = target_user_id;

  RETURN json_build_object('success', true);
END;
$$;
```

**IMPORTANTE**: Executar este SQL no Supabase SQL Editor antes de usar a funcionalidade de alterar senha.
